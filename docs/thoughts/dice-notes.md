# Dice — HEM Note

*(Shardhana Thoughts)*  
*Date: 2026-04-15*

---

## Starting Point

A question that started from a dice:

> Is the probability of the real world the same as the ideal probability?

- Ideal world → results are selected by probability
- Real world → results emerge from state change

HEM starts from this difference.

---

## FEM / AEM vs HEM

### FEM / AEM

Input → Calculation → Single result

- Same input always produces the same result
- Completely deterministic structure
- AEM can express complex behavior like fracture and collision,  
  but still calculates a single scenario result

### HEM

Input (initial state variables + Seed set) → State change → Result set → Statistical interpretation

- Input is not a single value, but a set of initial state conditions
- Micro-variations in initial conditions are included through Seed
- Results are not a single value — a result set is formed, from which range and distribution are derived

→ An interpretive structure that includes real-world uncertainty

---

## The Difference Through Dice

### Conventional approach (probability-based)

randint(1, 6)

Results are directly selected. Probability is assumed.

### HEM approach (state-based)

Initial condition (Seed) → Rotation · Collision · Friction → Stabilization → Face touching the ground = Result

- The result is not selected
- It is naturally determined at the end of state change

---

## The Role of Seed

Seed = subtle differences in initial conditions

- Same Seed → Same result (reproducible)
- Different Seed → Different result

Seed does not create the result —  
it is the starting state that determines the direction of the result.

---

## HEM Input/Output Structure

| | FEM / AEM | HEM |
|--|-----------|-----|
| Input | Fixed value | Initial state variables + Seed set |
| Output | Single value | Result set (Range) → Distribution |
| Interpretation | Single scenario | Multiple possibilities |
| Judgment | Whether criteria are met | Whether within safe range |
| Philosophy | Results are calculated | Results emerge |

---

## HEM Output Definition

The output of HEM consists of:

- Mean value
- Maximum / Minimum value
- Variance
- Confidence interval
- Probability of failure

---

## Core Philosophy

> Results are not "selected" — they "emerge."

> Probability is not a property of nature —  
> it is the way humans express uncertainty.

---

## One-line Definition

> Uncertain start → State change → Result set → Safe range judgment

> HEM does not deal with "one answer" —  
> it deals with "the range of possible outcomes in reality."

# 주사위 — HEM Note

*(Shardhana 생각창고)*  
*Date: 2026-04-15*

---

## 출발점

주사위에서 시작된 질문:

> 현실 세계의 확률은 이상적인 확률과 같은가?

- 이상적인 세계 → 확률로 결과를 선택한다
- 현실 세계 → 상태 변화로 결과가 발생한다

HEM은 이 차이에서 출발한다.

---

## FEM / AEM vs HEM

### FEM / AEM

입력 → 계산 → 단일 결과

- 동일 입력이면 항상 동일 결과가 나온다
- 완전한 결정론 구조다
- AEM은 파괴·충돌 등 복잡한 거동을 표현하지만  
  여전히 하나의 시나리오 결과를 계산한다

### HEM

입력(초기 상태 변수 + Seed 집합) → 상태 변화 → 결과 집합 → 통계적 해석

- 입력은 하나의 값이 아니라 초기 상태 조건 집합이다
- Seed를 통해 미세한 초기 변동이 포함된다
- 결과 집합으로부터 범위와 분포가 도출된다

→ 현실의 불확실성을 포함한 해석 구조

---

## 주사위로 보는 차이

### 기존 방식 (확률 기반)

randint(1, 6)

결과를 직접 선택한다. 확률을 가정한다.

### HEM 방식 (상태 기반)

초기조건(Seed) → 회전·충돌·마찰 → 안정화 → 바닥에 닿은 면 = 결과

- 결과는 선택되는 것이 아니다
- 상태 변화의 끝에서 자연스럽게 결정된다

---

## Seed의 역할

Seed = 초기 조건의 미세한 차이

- 동일 Seed → 동일 결과 (재현 가능)
- 다른 Seed → 다른 결과

Seed는 결과를 만드는 것이 아니라,  
결과의 방향을 결정하는 출발 상태다.

---

## HEM 입출력 구조

| 구분 | FEM / AEM | HEM |
|------|-----------|-----|
| 입력 | 고정값 | 초기 상태 변수 + Seed 집합 |
| 출력 | 단일 값 | 결과 집합 (Range) → 분포 |
| 해석 | 단일 시나리오 | 다중 가능성 |
| 판단 | 기준값 만족 여부 | 안전 영역 내 존재 여부 |
| 철학 | 결과를 계산한다 | 결과가 발생한다 |

---

## HEM 출력 정의

HEM의 출력은 다음과 같이 구성된다:

- 평균값
- 최대 / 최소값
- 분산
- 신뢰구간
- 실패 확률

---

## 핵심 철학

> 결과는 "선택"되는 것이 아니라 "발생"하는 것이다.

> 확률은 자연의 속성이 아니라,  
> 인간이 불확실성을 표현하는 방식이다.

---

## 한 줄 정의

> 불확실한 시작 → 상태 변화 → 결과 집합 → 안전 영역 판정

> HEM은 "정답 하나"가 아니라,  
> "현실에서 가능한 결과 범위"를 다루는 해석 방법이다.
