# Add simple graph visualization to the existing Shardhana GUI seed

We already have a minimal 1D HEM prototype in Python.

Current status:

* The simulation is rule-based
* No FEM equations are used
* No stiffness matrix is used
* The prototype currently prints state values as text in the terminal
* The simulation already shows localization at the center element

Now I want to extend the existing Shardhana GUI seed so that the simulation result can be visualized as a graph.

Goal:
Use the existing GUI window and add a simple plotting area that shows how the 1D HEM state evolves.

Requirements:

1. Keep the current 1D HEM calculation logic simple and rule-based.
2. Do not introduce FEM, stiffness matrices, or complex physics.
3. Use the existing GUI seed as the base.
4. Add input fields for:

   * number of elements (N)
   * time step (dt)
   * amplification factor (k)
   * number of steps (steps)
5. Add a Run button.
6. When Run is pressed:

   * execute the 1D HEM simulation
   * store the full history of state values at each step
7. Display a graph inside the GUI:

   * x-axis = element index
   * y-axis = state value
   * plot at least three curves:

     * Step 0
     * middle step
     * final step
8. Keep the GUI minimal and readable.
9. Keep the code beginner-friendly.
10. Do not introduce classes for the simulation logic yet unless needed for the GUI structure.
11. The simulation update should still use a copied list, not in-place overwrite.
12. Use matplotlib for plotting.

Simulation rule:

* Represent the 1D bar as a list of scalar state values

* Initialize all elements to 0.0

* Give the center element a small perturbation, for example 0.1

* For each interior element:

  * compute neighbor average

  * define:

    diff = old_state[i] - neighbor_avg

  * update using:

    new_state[i] = old_state[i] + dt * k * diff

* Store the state history for plotting
