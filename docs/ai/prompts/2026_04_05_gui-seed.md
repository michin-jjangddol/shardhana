# Shardhana GUI Seed 2026/04/05 — Minimal Split Window

This document defines the very first GUI seed for the Shardhana project.

The goal is not analysis yet.
The goal is to create the smallest possible PySide6 window
with a clear split layout that can grow over time.

---

## 1. Goal

Create a minimal desktop GUI using **PySide6**.

This is only the first seed of the program.

It should show:

- a left input panel
- a middle button area
- a right output panel

No plotting is needed.
No file save/load is needed.
No FEM/HEM calculation is needed yet.

The window only needs to establish the basic structure.

---

## 2. Requirements

### Framework
- Use **Python**
- Use **PySide6**

### Window title
- `Shardhana 1D FEM / HEM Demo`

### Layout
Use a simple horizontal split layout with 3 sections:

1. **Left panel — Input**
2. **Middle panel — Actions**
3. **Right panel — Output**

---

## 3. Left Panel (Input)

Add labeled input widgets for the following fields:

- Number of Nodes
- Number of Elements
- Total Length
- Young's Modulus (E)
- Area (A)
- Applied Force (F)

Use simple text input fields.
Default values may be inserted for testing.

---

## 4. Middle Panel (Actions)

Add two buttons:

- `Build Model`
- `Run Analysis`

For now:

- `Build Model` may print a simple placeholder message
- `Run Analysis` may print a simple placeholder message

No real analysis is required yet.

---

## 5. Right Panel (Output)

Add a read-only text output area.

This output area should display simple text logs such as:

- program started
- build model clicked
- run analysis clicked

Example output:

Program started.
Build Model clicked.
Run Analysis clicked.

---

## 6. Design Principles

Keep the code as simple as possible.

Important:

- Do not add unnecessary features
- Do not add graphics
- Do not add menus yet
- Do not add tabs yet
- Do not add save/load yet
- Do not add FEM logic yet

This is only a structural GUI seed.

---

## 7. Code Style

- Write clean and readable code
- Organize it in a single file for now
- Add short comments where useful
- Make sure the program runs directly

Preferred file name:

`main.py`

---

## 8. Expected Outcome

When executed, the program should:

- open a window
- show the three-panel structure
- allow text input
- respond to button clicks
- print simple messages in the output area

That is enough for version 0.1.

---

## 9. Future Expansion (Not for now)

This structure will later grow into:

- FEM 1D tension solver
- HEM mode
- model visualization
- file save/load
- menus and tools

But none of that should be implemented now.

---

## 10. Final Instruction

Please generate the simplest possible working PySide6 code
that satisfies the structure above.

Do not over-design it.
Do not add extra architecture.
Just make a clean, minimal, working first seed.
