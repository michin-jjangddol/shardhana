## A Linux-Inspired Architecture

Shardhana is built with inspiration from the Linux ecosystem.

Just as Linux keeps its kernel minimal and stable while allowing countless
distributions to flourish, Shardhana separates a lightweight **Core**
from flexible, optional **Plugins**.

The Core focuses on fundamental numerical methods and data structures.
Analysis features, design checks, code-specific logic, visualization,
and experimental ideas are added as plugins — only when needed.

Multiple core versions may coexist, allowing users to choose
between long-term stability and new experimentation.
By keeping the core small and the rest optional,
Shardhana aims to run anywhere —
from modern workstations to older, low-spec machines.

Shardhana does not force a single workflow.
Instead, it provides a solid foundation
on which each engineer can build their own.

## 리눅스 철학에서 출발한 샤드하나

Shardhana는 **리눅스의 설계 철학**에서 출발합니다.

리눅스가 최소·안정적인 커널 위에
다양한 배포판이 공존하듯,
Shardhana 역시 가벼운 **Core**와
선택 가능한 **Plugin**을 분리합니다.

Core는 수치해석의 본질에 집중하고,
해석·설계·기준 검토·시각화·실험적 기능은
필요할 때 플러그인으로 더해집니다.

Shardhana는 하나의 정답을 강요하지 않습니다.
안정적인 버전과 실험적인 버전이 함께 존재하며,
최신 장비부터 오래된 저사양 머신까지
각자의 환경에 맞게 사용할 수 있는 도구를 지향합니다.

단단한 기반 위에,
각 엔지니어의 방식이 자연스럽게 쌓이기를 바랍니다.
